several simplification sprints. The pitch under test:

**Auditor**: read-only audit (no code/test mutations). Conclusions are
based on grep sweeps, file inspection, and live boot of `pnpm dev`.

true`; the 503 branch is asserted by unit tests.)

### Sim D — Upload route without Cloudinary creds

```
POST /api/upload  →  HTTP 403  {"error":"Invalid CSRF token"}
```

**Verdict: ✅ correct gate order.** The handler checks CSRF *before* auth and
*before* storage availability. A real browser flow would echo the
`<prefix>-csrf` cookie via `x-csrf-token`; without it, the route refuses.
This is the expected outer ring of the chain. The `STORAGE_NOT_CONFIGURED`
503 branch is unit-tested at `route.test.ts`.
